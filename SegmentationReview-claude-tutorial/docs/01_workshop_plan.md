# Workshop plan

## Audience
Technical colleagues who already know Python and want to collaborate cleanly with GitHub.

## Duration
75 minutes.

## Agenda

### 1. Why Git and GitHub matter (10 min)
- Version history
- Collaboration without overwriting each other
- Reviewable changes
- Safer experimentation with branches
- Traceability for medical/research software work

### 2. Git capabilities map (10 min)
Walk through:
- local history
- branches
- remotes
- pull / fetch / push
- merge vs rebase
- PRs and review comments
- checks / CI
- Issues and templates

### 3. Hands-on branch and commit exercise (15 min)
Participants create their own feature branch and make a small change.

### 4. Pull request exercise (15 min)
They open a PR, request review, and respond to comments.

### 5. Rebase and conflict simulation (15 min)
One participant changes the same lines on `main`; another rebases and resolves the conflict.

### 6. LLM-assisted development hygiene (10 min)
Discuss commit quality, PR transparency, and review responsibility.

## Success criteria
Each participant:
- creates a branch
- makes 2 clean commits
- pushes a branch
- opens a PR
- leaves at least 1 review comment
- rebases once
- resolves 1 small conflict
