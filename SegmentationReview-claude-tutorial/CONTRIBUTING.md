# Contributing guidelines

## Workflow
1. Start from updated `main`
2. Create a feature branch
3. Keep changes focused
4. Open a PR early
5. Rebase on `main` before merge if needed

## Pull requests
A PR should:
- solve one problem
- have a clear title
- explain why the change exists
- describe how it was validated
- mention LLM assistance when materially used

## Review etiquette
- be specific
- be kind, direct, and technical
- separate blocking issues from suggestions
- prefer concrete examples over vague criticism
